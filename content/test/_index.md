## 1. 設計の根拠：トップ大学は何を見ているか

### 1-1. MIT（公式 "What We Look For"）

MITのAdmissions公式ページに明記されている評価基準：

| 評価基準 | 原文の要旨 |
|---|---|
| Alignment with MIT's mission | 科学・技術で世界をより良くする。「15歳までに感染症を根絶」する必要はない。一人の子に数学を教えることも世界を変える |
| Collaborative and cooperative spirit | MITの核心は協働。問題セットはグループで取り組む設計。一人で働くのが好きならMITは合わないかもしれない |
| Initiative | 機会を自ら掴みに行く。挑戦的な授業を取る、会社を始める、コミュニティに貢献する |
| Risk-taking | リスクを取り、失敗から学ぶ。失敗しても立ち直る回復力 |
| Hands-on creativity | 理論を実世界の問題に適用する。MITのモットーは "Mind and Hand"（頭と手） |
| Intensity, curiosity, and excitement | 量より質。少数のことに本気で取り組む |
| The ability to prioritize balance | 勉強だけではない。スポーツ、芸術、コミュニティ活動など |

**出典:** https://mitadmissions.org/apply/process/what-we-look-for/

### 1-2. MITエッセイプロンプト（2025-2026）

| プロンプト | 字数 | 問われていること |
|---|---|---|
| 1. なぜこの専攻をMITで学びたいか | 100語 | 知的好奇心、具体的な理由 |
| 2. 純粋に楽しんでやっていること | 150語 | 人柄、バランス |
| 3. 教育の中で期待と違うことをした経験 | 225語 | 独自性、イニシアチブ |
| 4. 他者と協力して学んだ・貢献した経験 | 225語 | 協働、コミュニティへの貢献 |
| 5. 予期しない状況にどう対処し、何を学んだか | 225語 | リスク対応、回復力、成長 |

**出典:** https://blog.collegevine.com/how-to-write-the-mit-application-essays!

### 1-3. Stanford

Stanfordが重視する4つの基準：

| 評価基準 | 内容 |
|---|---|
| Intellectual Vitality | Stanfordの最重要基準。教室の外でも学び続ける知的好奇心。独自の研究やプロジェクト |
| Impact and Leadership | 参加するだけでなく変化を生み出す。規模は問わない。自分の影響範囲で意味のある違いを作ったか |
| Personal Context and Resilience | 自分の状況の中でどう最善を尽くしたか。障害をどう乗り越えたか |
| Authenticity | 本物であること。大学受験のためにやったのではなく、本気で取り組んだか |

**出典:** https://www.goomi.ai/blog/stanford-admissions-guide

## 3. プログラム設計（マンツーマン伴走型）

### 基本情報

| 項目 | 内容 |
|---|---|
| 形式 | 隔週土曜、全5回、各2時間 |
| 時期 | 2026年5月〜7月 |
| 参加費 | 無料 |
| 定員 | 1〜5名（第1期） |
| 伴走 | 主催者が各参加者のメンターとして直接伴走 |

### セッション間の伴走

- セッション間に主催者と1on1（オンライン、30分程度×1-2回）
- 参加者が方向性を決め、主催者が実行を支援する（調査、開発補助、連絡先紹介等）
- 参加者の時間を最小限にしつつ、プロジェクトを前に進める

### カリキュラム

---

#### Day 1：自分の問いを見つける

**対応する評価基準:** Stanford: Intellectual Vitality / Authenticity、MIT: Initiative

| 時間 | 内容 |
|---|---|
| 0:00-0:20 | オリエンテーション。自己紹介。プログラムの目的：「5回で、自分の問いを見つけ、形にし、人に届ける」 |
| 0:20-0:50 | **自己理解ワーク。** 「自分が時間を忘れて没頭すること」「自分が怒りを感じること」「自分がこれまでで最も達成感を感じた瞬間」を書き出す。ここから「自分が本気で取り組みたい問い」を探す |
| 0:50-1:20 | **AIペルソナとの壁打ち。** 見つけた問いについて、AIに「この課題で困っている人」を演じさせて対話する。課題の解像度を上げる |
| 1:20-1:50 | **課題とソリューションの方向性を決める。** 「誰の、何の課題を、なぜ自分が解くのか」を1文で書く。さらに「どう解くか」の方向性を考える。プロダクトの形（アプリ、Webサービス、ツール等）を仮決めする |
| 1:50-2:00 | 次回までの進め方を相談 |

**セッション間：**
- 主催者と1on1で課題とソリューションを深掘り
- 可能であれば実際に困っている人に話を聞く（任意）
- プロトタイプの開発に着手（主催者が参加者の指示に基づいて開発の実行面を支援）

**なぜこの設計か：**
- Stanfordの "Intellectual Vitality" は「教室の外で自発的に学ぶ姿勢」。自分で課題を見つけるプロセスそのものがこれを体現する
- Stanfordの "Authenticity" は「本気で取り組んだか」。自分の原体験から課題を見つけることで、「なぜこれをやるのか」に嘘がなくなる
- 東大推薦の面接で「なぜこの課題か」と聞かれたとき、自分の言葉で答えられる

---

#### Day 2：作る

**対応する評価基準:** MIT: Hands-on creativity / Risk-taking、Stanford: Intellectual Vitality

| 時間 | 内容 |
|---|---|
| 0:00-0:20 | 各自の課題と解決策の方向性を共有 |
| 0:20-1:40 | **AIを使ってプロトタイプ開発。** 参加者が設計・意思決定し、AIに指示を出して実装する。主催者が技術面をサポート。「完璧でなくていい、動くものを作る」 |
| 1:40-2:00 | 進捗共有。次回までの開発計画を立てる |

**セッション間：**
- プロトタイプの開発を続ける。これがプログラム全体で最も重要な期間
- 主催者が参加者の指示に基づいて開発の実行面を支援（調査、コーディング補助、デザイン等）
- 参加者は「何を作るか」「どう改善するか」の意思決定に集中し、主催者が手を動かすことで参加者の時間を節約する
- 1on1を1-2回実施して方向性を確認

**なぜこの設計か：**
- MITの "Hands-on creativity" は「理論を実世界の問題に適用する」。実際に動くものを作ることがこれに直結する
- MITの "Risk-taking" は「不完全でも出す」。完璧主義を捨てて動くものを作る経験は、数オリ的な「正解を出す」思考の対極
- 主催者が実行面を支援することで、参加者は「何を作るか」の意思決定に集中できる。受験生の時間を節約しつつ、「自分が主導した」という実感は残る

---

#### Day 3：出して検証する

**対応する評価基準:** MIT: Risk-taking、Stanford: Impact、MIT Prompt 5（予期しない状況）

| 時間 | 内容 |
|---|---|
| 0:00-0:20 | プロトタイプのデモ。現状の共有 |
| 0:20-1:00 | **顧客検証。** 実際の人にプロダクトを使ってもらう（セッション内で外部の人をオンラインで招く、または参加者同士でテスト）。「これは本当に使いたいか？」「何が足りないか？」を聞く |
| 1:00-1:40 | **フィードバック分析。** 「想定と違った反応」を整理する。何が予想通りで、何が予想外だったか。方向転換が必要か |
| 1:40-2:00 | 改善方針を決める。次回までの計画 |

**セッション間：** フィードバックを反映して改善。主催者が支援。

**なぜこの設計か：**
- MIT Prompt 5は「予期しない状況にどう対処したか」。ユーザーの反応が想定と違う経験は、このプロンプトに直接使える素材になる
- Stanfordの "Impact" は「実際に変化を生み出したか」。実際のユーザーに使ってもらうことで、小さくても実際のインパクトが生まれる
- 東大推薦の差別化：「プロダクトを作って実際のユーザーに使ってもらった」はメダリストの中でも稀な経験

---

#### Day 4：磨く＋再検証

**対応する評価基準:** Stanford: Impact / Resilience、MIT: Risk-taking

| 時間 | 内容 |
|---|---|
| 0:00-0:20 | Day 3のフィードバックを踏まえた改善方針の共有 |
| 0:20-1:20 | **改善開発スプリント。** フィードバックを反映してプロダクトを改善する。主催者が技術面をサポート |
| 1:20-1:50 | **2回目の検証。** 改善版を再度人に使ってもらう。1回目と比べて反応がどう変わったかを確認する |
| 1:50-2:00 | Demo Dayの説明。審査基準の共有。最終調整の計画 |

**セッション間：** プロダクトの最終調整。Demo Day用のデモ準備（ピッチ資料は不要、デモを見せながら話す形式）。

**なぜこの設計か：**
- Build-Measure-Learnを2サイクル回せる（Day 3で1回目、Day 4で2回目）。1回の検証より2回の方が「改善した」という物語が強くなる
- Stanfordの "Resilience" は「フィードバックを受けて改善する力」。方向転換や改善の経験がこれに該当する
- ピッチ練習はDay 5の本番で十分。練習に1回使うより、プロダクトの質を上げる方が結果的にDemo Dayの説得力も上がる

---

#### Day 5：Demo Day

**対応する評価基準:** 全体の統合

| 時間 | 内容 |
|---|---|
| 0:00-0:15 | 最終準備 |
| 0:15-1:00 | **Demo Day。** 審査員の前で各参加者5分ピッチ + Q&A 5分。外部審査員（起業家、研究者等）を招く |
| 1:00-1:15 | 審査員協議 |
| 1:15-1:40 | **振り返り。** 「Day 1の自分と今の自分で何が変わったか」を各自が言語化する |
| 1:40-2:00 | **表彰 + クロージング。** 最優秀賞（ダボス会議派遣）、優秀賞等。アルムナイネットワーク開設。「次の一歩」の案内 |

**なぜこの設計か：**
- Demo Day＋表彰を維持することで、最優秀賞→ダボス派遣の構造を残す
- 振り返りの時間を明示的に設けることで、「変化の物語」を参加者自身が認識できる
- 外部審査員からのコメントは、推薦状や紹介に繋がる可能性がある

## 5. IMO金メダリスト不合格事例への回答

この学生がEdgeに参加していた場合、以下が変わっていた可能性がある：

1. **差別化:** 「IMO金メダリスト」に加えて「実世界の課題を見つけ、プロダクトを作り、ユーザーに届けた」という実績が加わる。他のメダリストとの差別化が明確になる
2. **面接:** 「数学で何を学びましたか」→「数学の力で実際にプロダクトを作り、人に使ってもらう経験をしました。そこで数学だけでは解けない問題があることを知りました」と答えられる。書類と実際のギャップがなくなる
3. **評定平均値:** Edgeでは解決できない。これは学校の成績の問題
4. **物語の一貫性:** 「数学が得意→数学で世界を変えたい→実際にやってみた→学んだ」という一貫した物語が作れる
